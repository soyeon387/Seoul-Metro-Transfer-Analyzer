#서울시 지하철 환승 난이도(피로도)를 반영한 편한 길 경로 추천 시스템

* 기존 지도 앱(최단 시간 중심)의 한계를 보완하여 환승 거리와 소요 시간을 기반으로 환승 피로도(Difficulty)를 점수화하고, 이를 반영한 최적 경로를 제안하는 알고리즘 개발

1. 주요 기술 스택 (Tech Stack)

* Language: Python
* Data Collection & Processing:
* requests: 공공데이터포털 API를 통한 실시간 환승 데이터 수집
* pandas, numpy: 대규모 지하철 역간 데이터 및 환승 데이터 전처리, 정규화

* Algorithm: 다익스트라(Dijkstra) 알고리즘 구현을 위한 우선순위 큐 관리

* Visualization:
-> matplotlib: 환승 난이도 상위 역사 시각화, 거리-시간 산점도, 경로별 누적 비용 그래프 구현



 2. 핵심 로직 및 알고리즘 (Logic)

1. 데이터 수집 및 난이도 산출:
* 서울교통공사 API를 통해 각 환승역의 환승 거리(m)와 환승 소요시간(초)을 수집
* 서로 단위가 다른 거리와 시간을 0~1 사이로 정규화한 뒤, 가중치를 적용해 환승 난이도 점수(0~100점)를 산출
* 수식: Difficulty = (0.5 * 정규화된 거리) + (0.5 * 정규화된 시간) * 100


2. 그래프 모델링:
* 노드(Node): 호선|역명 (예: "2|사당", "4|사당"은 서로 다른 노드로 취급)
* 간선(Edge) 가중치 설계:
* 호선 내 이동: 실제 열차 운행 시간 (분)
* 환승 이동: 기본 환승 시간(분) + (Alpha × 환승 난이도 점수)
* 여기서 ALPHA 값(설정값 0.2)을 조절하여 환승이 힘든 역을 얼마나 회피할지 결정


3. 경로 탐색 (Dijkstra Algorithm):
* Case 1 (최단 시간): Alpha = 0으로 설정하여 오직 시간만을 기준으로 탐색.
* Case 2 (편한 경로): Alpha = 0.2로 설정하여 환승 난이도가 높은 구간(예: 홍대입구, 노원 등)의 가중치를 높여 우회 경로를 탐색.



3. 문제 해결 방향

* 문제 인식: 홍대입구, 노원, 고속터미널 같은 역은 환승만 해도 지치는데 지도 앱은 무조건 빠르다고 추천한하는 것에 대한 불편함 인식
* 정량화의 어려움: 힘들다는 주관적 감각을 어떻게 수치화할 것인지
* 해결: 환승 거리와 소요 시간의 상관관계를 분석하고, 이를 정규화하여 Difficulty Score라는 객관적 지표로 변환

* 경로 최적화: 무조건 환승을 안 하는 것이 정답은 아님.
* 해결: ALPHA 파라미터를 도입해 조금 돌아가더라도 덜 걷는 경로와 힘들어도 빠른 경로 사이의 균형을 사용자가 선택할 수 있는 구조



4. 성과 및 의의 (Achievements)

* 데이터 기반 인사이트 도출: 분석 결과 홍대입구, 노원, 대림 등이 환승 난이도가 가장 높은 역으로 도출되었으며 이는 실제 이용객의 경험과 일치함
* 개인화된 경로 추천 가능성: 노약자, 임산부, 혹은 짐이 많은 여행객에게는 단순 시간이 아닌 이동 편의성이 중요함을 입증하고 대안 경로를 제시
* 시각적 입증: 경로별 누적 비용 그래프(거리-시간 산점도 등)를 통해 편한 경로가 심리적/신체적 비용을 얼마나 절감해 주는지 시각적으로 구현
